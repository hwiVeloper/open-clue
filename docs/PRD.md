# 제품 요구사항 문서 (PRD): OpenClue

> **"The Terminal-Based Escape Room Engine"**
> 터미널 프롬프트라는 제한된 환경에서 숨겨진 실마리(Clue)를 찾아 해방(Open)시킨다.

---

## 1. 제품 개요 (Product Overview)

**OpenClue**는 JSON으로 작성된 시나리오 파일을 로드하여 터미널 환경에서 텍스트 기반 방탈출 게임을 플레이할 수 있는 오픈소스 CLI 엔진이다.

실제 방탈출의 핵심 기믹(공간 탐색, 아이템 획득, 퍼즐 풀기, 문 열기)을 순수 텍스트 명령어로 구현하며, 시나리오 데이터는 AES-256 암호화를 통해 플레이 전까지 내용이 노출되지 않도록 보호된다.

---

## 2. 핵심 가치 (Core Values)

| 가치 | 설명 |
|:---|:---|
| **몰입감** | 그래픽 없이 텍스트와 ASCII 아트만으로 높은 몰입감을 제공하는 TUI 구현 |
| **확장성** | 누구나 JSON으로 시나리오를 작성하고 암호화하여 공유 가능 |
| **보안성** | 시나리오 전체 AES-256 암호화 + 정답 SHA-256 해싱으로 스포일러 원천 차단 |
| **간결함** | Python 미설치 환경에서도 실행 파일 하나로 즉시 플레이 가능 |

---

## 3. 사용자 페르소나 (User Personas)

### Player (플레이어)
- 텍스트 게임 또는 방탈출을 즐기는 사용자
- 터미널 환경에 거부감이 없는 개발자 또는 레트로 게임 팬
- 암호화된 `.dat` 파일을 받아 엔진 하나로 즉시 플레이

### Creator (시나리오 제작자)
- 자신만의 스토리를 게임화하고 싶은 창작자
- JSON 문법을 작성할 수 있는 수준의 기술 이해도 보유
- 빌더 CLI를 이용해 JSON을 암호화하여 배포

---

## 4. 기능 요구사항 (Functional Requirements)

### 4.1 게임 엔진 (Player Mode)

| ID | 요구사항 | 우선순위 |
|:---|:---|:---|
| FR-01 | 암호화된 `.dat` 파일을 복호화하여 메모리에 로드 | Must |
| FR-02 | 현재 위치(Room), 인벤토리, 플래그(Flag) 상태 실시간 유지 | Must |
| FR-03 | 텍스트 명령어 입력 파서 구현 (`look`, `inspect`, `use`, `inv`, `hint`) | Must |
| FR-04 | Rich 라이브러리 기반 TUI 렌더링 (패널, 테이블, 컬러) | Must |
| FR-05 | 퍼즐 정답 입력 시 SHA-256 해시 비교 검증 | Must |
| FR-06 | 조사 지점(Point) 별 조건부 상호작용 (아이템 보유 여부, 플래그 상태) | Must |
| FR-07 | 아이템 획득 및 사용 로직 | Must |
| FR-08 | 방 이동(Room Navigation) 로직 | Must |
| FR-09 | 타임어택 퍼즐 지원 (제한 시간 내 입력) | Should |
| FR-10 | ASCII 아트 이미지 힌트 출력 | Should |
| FR-11 | 효과음 재생 (playsound, 이벤트 트리거 시) | Could |

### 4.2 시나리오 빌더 (Builder Mode)

| ID | 요구사항 | 우선순위 |
|:---|:---|:---|
| FR-20 | 원본 JSON 파일의 스키마 유효성 검사 (Pydantic) | Must |
| FR-21 | 방-포인트-아이템 간 참조 ID 무결성 검사 | Must |
| FR-22 | 정답 평문을 SHA-256 해시값으로 자동 변환 | Must |
| FR-23 | JSON 전체를 AES-256 (GCM Mode)으로 암호화하여 `.dat` 파일 생성 | Must |
| FR-24 | 이미지 파일을 `.dat`에 바이너리로 번들링 | Should |

### 4.3 게임 기믹 (Game Mechanics)

| 기믹 | 설명 |
|:---|:---|
| **조사 (Inspect)** | 포인트 조사 시 묘사 텍스트 출력 또는 아이템 획득 |
| **퍼즐 (Puzzle)** | 텍스트 정답 입력 → 해시 비교 → 성공 시 액션 실행 |
| **아이템 조건** | 특정 아이템이 인벤토리에 있어야 상호작용 가능 |
| **플래그 조건** | 시스템 상태(`is_power_on` 등)에 따라 분기되는 로직 |
| **방 이동** | 퍼즐 성공 또는 조건 충족 시 다른 Room으로 이동 |
| **해금 (Unlock)** | 특정 포인트나 아이템이 조건 충족 후 활성화 |

## 5. 비기능 요구사항 (Non-Functional Requirements)

| 항목 | 요구사항 |
|:---|:---|
| **성능** | 시나리오 로딩 및 명령어 응답 0.1초 이내 |
| **이식성** | Windows, macOS, Linux 터미널에서 동일한 UI 레이아웃 |
| **오프라인** | 서버 연결 없이 모든 로직이 로컬에서 완결 |
| **배포** | Python 미설치 환경에서도 단일 실행 파일로 동작 |
| **보안** | 시나리오 파일은 암호화되어 평문으로 절대 읽힐 수 없어야 함 |

## 6. CLI 인터페이스 설계

```bash
# 게임 플레이
clue play <scenario.dat>

# 시나리오 빌드 (JSON → 암호화 .dat)
clue build <scenario.json>

# 시나리오 유효성 검사만 수행
clue verify <scenario.json>
```

### 인게임 명령어

| 명령어 | 기능 |
|:---|:---|
| `look` / `ls` | 현재 방 묘사 및 조사 가능 지점 목록 출력 |
| `inspect <point>` / `cd <point>` | 특정 지점 조사 또는 퍼즐 진입 |
| `inv` | 현재 소지 아이템 목록 확인 |
| `use <item>` | 아이템 사용 |
| `hint` | 제작자가 설정한 힌트 출력 |
| `help` | 사용 가능한 명령어 목록 출력 |
| `quit` | 게임 종료 |

## 7. 유저 플로우 (User Journey)

```
시작: clue play my_room.dat
  │
  ▼
[복호화] → 시나리오 메모리 로드
  │
  ▼
[현재 방 출력] look → 방 묘사 + 조사 가능 포인트 목록
  │
  ▼
[탐색] inspect desk → 책상 조사 → 메모지 획득
  │
  ▼
[퍼즐] inspect safe → 금고 퍼즐 → 정답 입력 → 해시 검증
  │
  ├─ 정답 O → 열쇠 획득 + 플래그 업데이트
  └─ 정답 X → 오류 메시지 + 재시도
  │
  ▼
[이동] use key → 문 열림 → 다음 방으로 이동
  │
  ▼
[클리어] 탈출 성공 메시지 + 소요 시간 출력
```
