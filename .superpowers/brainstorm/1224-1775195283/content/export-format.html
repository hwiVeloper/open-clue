내보내기 — 어떤 형식으로?

검증 통과 후 마지막 단계에서 어떤 출력을 줄지 결정합니다

A

JSON + DAT 동시 저장

내보내기 버튼 하나로 scenario.json(편집용)과 scenario.dat(배포용)를 동시에 저장. 선택지 없이 항상 둘 다.

  • 가장 단순한 UX — 고민할 필요 없음
  • JSON이 노출되는 게 싫은 배포자에겐 불편할 수 있음
B

선택 저장 (JSON / DAT / 둘 다)

내보내기 화면에서 체크박스로 원하는 형식을 선택. 개발 중엔 JSON만, 배포 시엔 DAT만, 아니면 둘 다.

  • 유연하지만 한 단계 더 생김
  • 대부분의 사용자는 "둘 다"를 고를 가능성 높음
C

JSON 저장 후 빌드는 별도 명령

빌더는 .json만 저장. DAT 변환은 기존 clue build 명령으로 따로. 빌더와 빌드를 분리.

  • 역할이 명확히 분리됨
  • 배포까지 두 단계 필요
